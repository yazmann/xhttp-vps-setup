# Безопасность

## Сообщить об уязвимости

Не публикуйте в Issues, Pull Requests или комментариях пароли, API-токены, IP-адреса серверов, домены, ссылки подписок и полный текст конфигурации.

После включения публичного репозитория используйте **Private vulnerability reporting** в GitHub. Если этот канал ещё не включён, создайте минимальный публичный Issue без технических деталей и укажите, что требуется приватная связь.

## Поддерживаемая версия

Поддерживается только актуальная версия ветки `main`.

## Что проверить перед отчётом

- используется чистый Ubuntu 22.04+;
- скрипт запускался от `root`;
- домен указывает на IPv4 VPS;
- в сообщении нет секретных данных.
