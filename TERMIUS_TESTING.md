# Проверка установщика через Termius

## До подключения

1. Переустановите VPS на чистую Ubuntu 22.04, 24.04 или 26.04.
2. Создайте DNS `A`-запись домена на IPv4 VPS.
3. Если используется Cloudflare, включите **DNS only** (серое облако) и удалите `AAAA`-запись.
4. Во внешнем firewall хостинга разрешите текущий SSH-порт, TCP/80 и TCP/443. Случайный порт панели также должен быть доступен; его значение появится в итоговом отчёте. На большинстве обычных VPS внешнего firewall нет.

## Загрузка через Termius

1. Создайте Host в Termius: IP сервера, пользователь `root`, пароль или SSH-ключ.
2. Подключитесь и убедитесь, что приглашение выглядит как `root@...#`.
3. Откройте SFTP в Termius и загрузите локальный файл `install-xhttp-vps.sh` в каталог `/root`.
4. В терминале выполните:

```bash
ls -lh /root/install-xhttp-vps.sh
chmod 700 /root/install-xhttp-vps.sh
bash -n /root/install-xhttp-vps.sh
```

Команда `bash -n` не должна вывести ошибок.

## Первый тест: самостоятельный сервер

Запустите:

```bash
/root/install-xhttp-vps.sh
```

Для первого контрольного прогона выберите:

1. `Standalone VPN server`.
2. Введите подготовленный домен.
3. Задайте имя, которое должно отображаться в VPN-клиентах.
4. На вопрос о WARP ответьте `n`. WARP лучше проверить отдельным вторым прогоном после подтверждения базовой работы.
5. Для дизайна заглушки выберите `1` — автоматический вариант.
6. Подтвердите установку буквой `y`.

Не закрывайте Termius во время обновления Ubuntu. Скрипт может работать несколько минут и самостоятельно ждёт освобождения блокировок `apt`.

## Ожидаемый результат

В финальном аудите должны получить `OK`:

- Ubuntu package upgrade;
- BBR + fq;
- IPv6 disabled;
- DNS, UFW и TLS;
- 3x-ui и NGINX;
- public self-steal website;
- XHTTP Reality self-steal;
- первый VLESS-клиент;
- обычная подписка;
- маршрутизация HAPP + INCY;
- Mihomo subscription.

Последняя строка должна быть:

```text
RESULT: ALL CHECKS PASSED
```

Ниже будут выведены:

- адрес панели, логин и пароль;
- подписка для HAPP/INCY;
- отдельная подписка для Mihomo;
- путь к защищённому файлу с результатом.

Повторно показать реквизиты можно командой:

```bash
cat /root/xhttp-vps-result-*.txt
```

## Проверка с устройства

1. Откройте `https://ВАШ-ДОМЕН` в обычном браузере — должен открыться сайт-заглушка, а не панель 3x-ui.
2. Добавьте `HAPP / INCY SUBSCRIPTION` в соответствующее приложение. Маршрутизация RoscomVPN должна прийти вместе с подпиской.
3. Для Clash Meta, Clash Verge или другого Mihomo-клиента используйте отдельный адрес `MIHOMO SUBSCRIPTION`.
4. Обновите подписку, выберите созданный профиль и включите VPN.
5. Откройте `https://ipinfo.io` до и после подключения. После подключения публичный IP должен измениться на IP VPS.

## Проверка после перезагрузки

Если итоговый отчёт рекомендует reboot:

```bash
reboot
```

Через минуту повторно подключитесь в Termius и выполните:

```bash
systemctl is-active x-ui nginx
ss -lntp | grep -E ':443|x-ui|nginx'
sysctl net.ipv4.tcp_congestion_control net.core.default_qdisc
sysctl net.ipv6.conf.all.disable_ipv6
```

Ожидаются `active`, Xray на TCP/443, `bbr`, `fq` и значение IPv6 `1`.

## Если установка прервалась

Не запускайте основной установщик повторно поверх установленной панели. Загрузите через SFTP файл `finish-xhttp-vps.sh` в `/root`, затем выполните:

```bash
chmod 700 /root/finish-xhttp-vps.sh
/root/finish-xhttp-vps.sh
```

Для диагностики:

```bash
systemctl status x-ui nginx --no-pager --full
journalctl -u x-ui -n 100 --no-pager
nginx -t
```

Скопируйте весь блок с первой строкой `ERROR` и результат этих команд.

## Второй тест: нода

Режим `Node for an existing 3x-ui panel` проверяйте после успешного самостоятельного сервера на новой чистой установке Ubuntu. Он не создаёт локального пользователя и не внедряет настройки HAPP/INCY/Mihomo: пользователей и подписки контролирует основная панель. В результате он выдаёт Panel URL, логин, пароль и API Token для добавления в `Nodes`.
