# Тестирование установщика на чистом VPS

Тестовый режим предназначен для многократной проверки установки без запросов к боевому Let's Encrypt и без расходования лимита сертификатов.

## Перед запуском

- Используйте чистую Ubuntu 22.04, 24.04 или 26.04.
- DNS-запись `A` домена должна указывать на IPv4 нового VPS.
- В Cloudflare запись должна быть в режиме **DNS only** (серое облако).
- Удалите запись `AAAA`, если она существует.
- Подключитесь к серверу через Termius как `root`.

## Передача и запуск скрипта через Termius

В Termius откройте SFTP, перейдите в `/root` и загрузите файл `install-xhttp-vps.sh`.

Затем выполните в терминале:

```bash
chmod 700 /root/install-xhttp-vps.sh
/root/install-xhttp-vps.sh
```

## Что выбрать

1. Выберите `Standalone VPN server` для проверки самостоятельного VPN-сервера либо `Node` для проверки ноды.
2. В разделе сертификата выберите `Test — self-signed certificate`.
3. Введите домен, уже направленный на VPS.
4. Укажите отображаемое название VPN.
5. Выберите WARP и вариант сайта-заглушки.

Тестовый режим создаёт локальный самоподписанный сертификат на 30 дней. Он не обращается к Let's Encrypt и поэтому не попадает под его лимиты.

## Ожидаемый результат

В конце должна появиться строка:

```text
RESULT: ALL CHECKS PASSED
TLS MODE: test
```

Также должны получить статус `OK`:

- обновление Ubuntu и обязательные пакеты;
- BBR + fq;
- отключение IPv6;
- DNS и UFW;
- тестовый TLS-сертификат;
- панель 3x-ui и её API;
- nginx и сайт self-steal;
- XHTTP + REALITY inbound на порту 443;
- первый клиент и подписки в режиме самостоятельного сервера;
- правила HAPP/INCY и endpoint Mihomo;
- WARP, если он был выбран.

В терминале будут показаны адрес панели, логин, пароль и готовые адреса подписок. Копия результата сохраняется в `/root/xhttp-vps-result-*.txt`.

## Важное ограничение тестового режима

Серверная часть и содержимое подписок проверяются полностью, но браузер, HAPP, INCY или Mihomo могут отклонить подключение к HTTPS из-за недоверенного самоподписанного сертификата. Это ожидаемо и не означает ошибку установки.

Для финальной проверки на телефоне переустановите чистый VPS или удалите тестовую установку, снова запустите скрипт и выберите `Production — trusted Let's Encrypt certificate`. Боевую установку следует запускать после снятия лимита Let's Encrypt либо с другим тестовым поддоменом.

## Если тест завершился ошибкой

Скопируйте последние строки терминала и выполните:

```bash
systemctl status x-ui --no-pager
systemctl status nginx --no-pager
journalctl -u x-ui -n 100 --no-pager
cat /root/xhttp-vps-result-*.txt 2>/dev/null
```

Для максимально чистого повторного теста рекомендуется переустановить VPS, а не продолжать поверх частично выполненной установки.
