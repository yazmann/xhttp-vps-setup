# Автоматическая установка XHTTP VPN-сервера или 3x-ui ноды

Основной файл: `install-xhttp-vps.sh`.

> Это независимый проект автоматизации. Он не является разработкой и не аффилирован с 3x-ui, Xray/XTLS или Cloudflare. Скрипт устанавливает и настраивает сторонние компоненты, права на которые принадлежат их авторам. Подробности: [THIRD_PARTY_NOTICES.md](THIRD_PARTY_NOTICES.md).

Скрипт предназначен для чистой Ubuntu VPS и предлагает два режима: самостоятельный VPN-сервер либо нода для уже существующей 3x-ui панели. Поддерживаются Ubuntu 22.04, 24.04 и 26.04; для более новых выпусков выполняется осторожный режим проверки. После определения версии установщик обновляет индексы репозиториев и все текущие пакеты Ubuntu, затем проверяет архитектуру, systemd, доступность обязательных пакетов и фактическое появление необходимых команд. Если обновлённые системные компоненты требуют перезагрузки, это будет явно указано в итоговом отчёте.

Он спрашивает домен, отображаемое имя VPN, необходимость WARP и дизайн сайта-заглушки. Техническое имя сервера определяется автоматически, а системный часовой пояс Ubuntu не изменяется. Доступны пять профессионально оформленных тематик и стабильный автоматический выбор по домену. Каждая тема использует отдельную реальную фотографию с бесплатной лицензией Unsplash и подписью автора. Фото скачивается на VPS при установке; после развёртывания сайт не зависит от внешнего CDN. Внешние шрифты и аналитика не используются.

IP основной панели не запрашивается, не сохраняется и не выводится. Внешний IP VPS определяется только для проверки DNS; все итоговые адреса панели и подписок используют домен.

Во время автоматической настройки web-панель временно слушает только `127.0.0.1`. Скрипт использует выданный официальным установщиком API Token и локальные запросы с `Authorization: Bearer`, поэтому создание inbound, клиента и подписок не зависит от входа через домен, логина или session cookie. Внешний IPv4-доступ к панели включается только после завершения и проверки конфигурации.

Свободные нестандартные порты, URI, логин и пароль генерируются автоматически.

Автоматически настраиваются BBR + `fq`, полное отключение IPv6, 3x-ui, Let's Encrypt, UFW, nginx self-steal, приватный сервер подписок, готовый `VLESS + XHTTP + REALITY` inbound на TCP/443 и, при выборе, маршрутизация `*.ru`/`geoip:ru` через встроенный WARP. В самостоятельном режиме создаётся первый клиент, в подписки HAPP и INCY внедряются актуальные правила [RoscomVPN Routing](https://github.com/hydraponique/roscomvpn-routing), а для Clash/Mihomo включается отдельный YAML endpoint. Режим ноды не меняет клиентские настройки подписок: ими управляет основная панель.

Финальный аудит проверяет сохранённые параметры inbound, self-steal target/SNI, наличие клиента, получение подписки через публичный порт 443, заголовок HAPP, профиль INCY и YAML Mihomo. Если любая обязательная проверка выбранного режима не пройдена, установка не считается успешной.

Стартовое меню предлагает самостоятельный VPN-сервер, ноду для существующей панели, полное удаление созданной конфигурации или выход. При удалении используются сохранённые исходные значения: сетевые sysctl возвращаются назад, созданные службы/файлы/сертификат/UFW-правила удаляются, а пакеты удаляются только если до запуска скрипта их не было.

## Запуск с компьютера

```powershell
scp .\install-xhttp-vps.sh root@IP_НОВОГО_VPS:/root/
ssh root@IP_НОВОГО_VPS
chmod 700 /root/install-xhttp-vps.sh
/root/install-xhttp-vps.sh
```

До запуска создайте DNS A-запись домена на IP нового VPS. Для Cloudflare нужен режим **DNS only** (серое облако), без AAAA-записи.

В режиме самостоятельного сервера итоговый отчёт выдаёт готовую подписку для первого клиента. В режиме ноды отчёт выдаёт Panel URL и реквизиты, которые нужно добавить в раздел `Nodes` основной панели.

## Запуск напрямую из GitHub

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/yazmann/xhttp-vps-setup/main/install-xhttp-vps.sh)
```

Фотографии ожидаются в каталоге `assets/` публичного репозитория. Если нужного файла там пока нет, установщик автоматически использует оригинал Unsplash. Базовый адрес можно переопределить переменной `REPO_ASSET_BASE`.

Если основная установка дошла до панели/nginx, но прервалась на настройке API, скопируйте `finish-xhttp-vps.sh` на VPS и запустите от root. Он найдёт сохранённый `/root/3xui-vps-*.env`, завершит inbound, подписки/WARP и выведет итоговый аудит с реквизитами.

Пошаговый контрольный прогон через Termius описан в [TERMIUS_TESTING.md](TERMIUS_TESTING.md).
